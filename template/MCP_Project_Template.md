# MCP Project Template

## Executive Purpose
Describe why the project exists and what capability it proves. Summarize who it is for
and what problem it solves in 2–4 sentences.

## Domain & Knowledge Area
Explain the industry, users, workflows, and key technologies or systems involved.

## Problems in the Domain
List the main operational, architectural, and business issues this project addresses.
Connect them directly to responsibilities and pain points from the job description.

## Core MCP + AI Solution
Describe the primary idea, target users, and value. Explain how MCP tools and AI
collaborate to solve the problems above.

## MCP Architecture
Detail the MCP server responsibilities, tool set, data sources, and AI agent behavior.
Include how tools are composed, any external APIs or systems, and error/safety
considerations.

## Additional MCP Project Ideas
List 3–5 related MCP project variations that reuse the same domain knowledge or stack
in different ways (e.g., observability, automation, reporting, self-service).

## AI Optimization
Explain how AI improves MTTR, automation, decision quality, or developer velocity.
Mention model usage patterns, prompt strategies, and any feedback loops.

## Technical Stack
List languages, infrastructure, telemetry, storage, security, and integration
technologies. Emphasize overlap with the job description stack.

## Trade-offs
Describe key trade-offs, risks, constraints, and plausible alternatives. Call out
scaling, reliability, cost, and data/safety concerns.

## Project Management
Outline scope, roadmap phases, and success metrics. Include assumptions and any
dependencies or prerequisites.

## Portfolio Value
Explain why this project demonstrates senior capability for this specific job
description. Map sections explicitly to responsibilities and requirements from the JD.
